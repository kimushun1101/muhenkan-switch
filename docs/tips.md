## 設定画面のキーボードショートカット

関連: 設定 GUI, Ctrl+S, Ctrl+Q, F1, Escape, ショートカット

設定ウィンドウで使えるショートカット。アプリ内ヘルプ (`F1`) と同じ内容です。

| キー | 機能 |
|---|---|
| `Ctrl` + `S` | 設定を適用（保存） |
| `Ctrl` + `Q` | アプリを終了 |
| `F1` | ヘルプウィンドウを表示 |
| `Escape` | 設定画面をトレイに格納 / ヘルプを閉じる |

## 無変換 + キーの早見表

関連: 無変換キー, kanata, dispatch, ホームポジション, レイヤー

固定割り当て（`muhenkan.kbd` で定義）：

| キー | 機能 |
|---|---|
| `H` / `J` / `K` / `L` | ←/↓/↑/→（カーソル移動） |
| `U` / `I` | 単語単位の左/右移動 |
| `Y` / `O` | Home / End |
| `N` / `M` | BackSpace / Delete |
| `;` | Escape |
| `V` | プレーンテキスト貼付 / エクスプローラーでは更新日時でリネーム |
| `C` | タイムスタンプ入力 / エクスプローラーではタイムスタンプ付きで複製 |
| `X` | エクスプローラーでファイル名からタイムスタンプを除去 |
| `Z` | タイムスタンプ位置を切替（前 ↔ 後） |
| `,` / `.` | 句読点切替（設定で 4 パターン） |
| `F1` | 設定ウィンドウを最前面に表示 |

カスタマイズ可能な割り当て（GUI または `config.toml` で変更）：

| キー | 種別 | デフォルト |
|---|---|---|
| `1`–`5` | フォルダ | Home / Desktop / Documents / Downloads / ~/proj |
| `Q` / `W` / `E` / `R` / `G` | 検索 | ChatGPT / Google 翻訳 / Weblio 英和 / Weblio 類語 / Google |
| `A` / `S` / `D` / `F` / `T` | アプリ | VS Code / Slack / OneNote / Firefox / Terminal |

詳細は [docs/customize.md — デフォルトのキー割り当て](customize.md#デフォルトのキー割り当て)、配置の理由は [docs/design.md — キー割り当ての設計思想](design.md#キー割り当ての設計思想) を参照。

## よく使うシナリオ

関連: ワークフロー, ユースケース, 効率化

### コードや Web 記事の書式を取り除いて貼り付けたい
- 操作: コピーしたあと `無変換` + `V`
- 補足: 通常の `Ctrl` + `V` と違いクリップボードをプレーンテキスト化して貼り付けます。Slack やメモアプリで書式持ち込みを避けたい場面で。

### 選択したテキストを Web 検索したい
- 操作: テキストを選択 → `無変換` + `Q` / `W` / `E` / `R` / `G`
- 補足: GUI の「検索」タブで URL とキーを編集できます。URL 内の `{query}` 位置に選択テキストが埋め込まれます。

### よく使うアプリを最前面に持ってきたい
- 操作: `無変換` + `A` / `S` / `D` / `F` / `T`
- 補足: 起動済みなら最前面化、未起動なら起動します。GUI の「アプリ」タブで割り当てを変更可能。

### よく使うフォルダを瞬時に開きたい
- 操作: `無変換` + `1`–`5`
- 補足: GUI の「フォルダ」タブで割り当てを変更可能。

### ファイル名にタイムスタンプを付けたい
- 操作: テキスト入力欄で `無変換` + `C`
- 補足: フォーマット（`yyyy-MM-dd` 等）と位置（前/後）は GUI の「タイムスタンプ」タブで設定。`無変換` + `Z` で前/後を即時トグル。

### エクスプローラー上のファイル名を更新日時に揃えたい
- 操作: ファイルマネージャで対象ファイルを選択 → `無変換` + `V`
- 補足: ファイル更新日時をタイムスタンプとしてリネームします。`無変換` + `C` で複製、`無変換` + `X` でタイムスタンプ除去。

## トラブルシューティング

関連: 動かない, kanata, 権限, Wayland, 再起動

### 無変換キーが反応しない
- kanata プロセスが起動しているか確認（トレイアイコンの状態表示）
- Linux: `uinput` グループ追加と udev ルール設定が済んでいるか確認（[docs/setup.md — Linux の追加設定](setup.md#linux-の追加設定)）
- ハードウェアキーボードに無変換キーがあるか（US 配列キーボードでは使えません）

### アプリ切り替え・フォルダオープンが効かない
- Linux: `xdotool` / `wmctrl` が必要（`sudo apt install xdotool wmctrl`）
- Wayland セッションでは未対応（[Issue #105](https://github.com/kimushun1101/muhenkan-switch/issues/105)）。X11 セッションに切り替えてください

### 設定を変更したのに反映されない
- GUI で変更し `Ctrl` + `S` で保存しただけでは kanata に反映されません。GUI のメニューから kanata を再起動してください
- `config.toml` / `muhenkan.kbd` を直接編集した場合も同様に kanata 再起動が必要

### タイムスタンプ入力やファイル名変更が動かない（Linux）
- `xclip` / `xprop` が必要（[docs/setup.md — Linux の外部ツール依存](setup.md#linux-の外部ツール依存)）
- Wayland では未対応

## FAQ

関連: 質問, 仕様, 設計

### なぜ無変換キーを選んだのか
押しやすい位置にありながらほぼ使われていないキーのため。詳細な選定基準は [docs/design.md — キー割り当ての設計思想](design.md#キー割り当ての設計思想) を参照。

### macOS で使える？
JIS 配列向けの設定ファイル (`muhenkan-macos.kbd`) は同梱していますが**動作未検証**です。報告・PR 歓迎。詳細は [docs/setup.md — macOS をお使いの方へ](setup.md#macos-をお使いの方へ)。

### 起動キーを無変換以外（変換キー等）に変更したい
`muhenkan.kbd` の編集が必要です。手順は [docs/customize.md — 起動キーを変更する](customize.md#1-起動キーを変更する) を参照。

### `Ctrl` + `Z` や `Ctrl` + `F` を無変換レイヤーに含めないのはなぜ？
OS 標準ショートカットで足りるものは含めない方針です。詳細は [docs/design.md — キー割り当ての設計思想](design.md#キー割り当ての設計思想)。

### CapsLock を Ctrl にしたい
muhenkan-switch では提供していません。OS 標準機能で行います。手順は [docs/customize.md — CapsLock → Ctrl リマップ](customize.md#capslock--ctrl-リマップ)。

### ログイン時の自動起動で設定ウィンドウが表示されない
仕様です。自動起動時はトレイ常駐 + kanata 起動のみを行い、設定ウィンドウは表示しません。設定を開くにはトレイアイコンをクリックするか「設定...」を選択してください（手動起動時は従来どおりウィンドウが表示されます）。
